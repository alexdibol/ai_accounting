AI for Audit & Accounting

A governance-first, educational guide for U.S. audit and accounting practice — built as an executable learning system: book + Colab notebooks. The objective is not speed or automation, but professional defensibility, traceability, and auditability under GAAS, PCAOB, SOX/ICFR, and U.S. tax practice constraints.

How to Use This Book + Notebooks

1) Read with a professional goal

Each chapter is written for real audit/accounting settings: what the model can do today, where it fails, and what “good output” means under supervision, documentation, and professional skepticism.

Mental model Failure modes Good output

2) Run the Colab notebook

Each chapter has a Colab notebook that operationalizes the chapter: structured inputs/outputs, explicit assumptions, and governance-native artifacts (manifests, logs, risk registers, versioned deliverables).

Colab Artifacts Auditability

3) Keep the verification posture

Treat outputs as drafts unless verified. Separate facts from assumptions. Keep open items explicit. Default to “Not verified” until evidence and human review exist.

Traceability Assumptions Accountability

Suggested path (fast)

  • Start: Level 1 to learn disciplined drafting (facts vs assumptions, “Not verified”).
  • Then: Level 3 to see checkpointed workflows + immutable logs in action.
  • Finally: Level 5 to understand organization-grade intake, QA, sign-off, and archiving.

Suggested path (full)

  • 1 → 2: drafting + structured reasoning (issue spotting, alternatives, exception hypotheses)
  • 3: agentic workflows with human checkpoints and artifact trails
  • 4 → 5: reusable assets, evaluation harnesses, and end-to-end operating model controls
